Namespace filter — which approach?

Three ways to replace the free-text input. All three fetch distinct namespaces from the backend on page load.

A — Plain dropdown

Swap the text input for a <select>. Pick from a list, no typing. Simple, zero dependencies. Can't search if there are many namespaces.

Pros

  • Minimal code
  • No ambiguity

Cons

  • No type-to-filter
  • Awkward with 20+ namespaces

B — Native datalist

Keep the text input, attach an HTML5 <datalist>. Browser shows suggestions as you type. Zero JS, zero dependencies — but browser styling varies and you can't style the dropdown.

Pros

  • Type-to-filter built in
  • No library needed
  • Still allows free text

Cons

  • Inconsistent across browsers
  • No control over dropdown style

C — Custom combobox

Text input + a custom-rendered dropdown that filters as you type. Consistent styling, keyboard navigable, matches the rest of the UI. Small amount of state to manage in React.

Pros

  • Polished, consistent UI
  • Fully styleable
  • Keyboard support

Cons

  • ~60 lines of React state
  • Click-outside to close logic
