Namespace combobox — design

Two parts: a new backend endpoint + a new React component. Does this look right?

Backend — new endpoint

GET /api/namespaces returns a sorted list of distinct namespace strings from the local DB.

SELECT DISTINCT namespace FROM executions ORDER BY namespace
→ ["company.finance", "company.ops", "company.team"]

Added to ExecutionRepository + a new GET /api/namespaces route in ExecutionResource. No new service class needed.

Frontend — two new files

useNamespaces.ts
React Query hook — fetches /api/namespaces once, cached. Returns string[].
NamespaceCombobox.tsx
Props: value, onChange. Replaces the plain <input> in ExecutionListPage.

Combobox interaction states

Idle (nothing typed)

Dropdown closed. Table shows all.

Typing "comp"
company.finance
company.ops
company.team

Dropdown open. Table already filtering on "comp".

Hovering a suggestion
company.finance
company.ops
company.team

Arrow keys or mouse highlight. Enter or click selects.

After selecting "company.ops"

Dropdown closes. Table filtered to exact namespace.

Keyboard behaviour

↓ / ↑Move highlight through suggestions EnterSelect highlighted suggestion EscapeClose dropdown, keep current text Click outsideClose dropdown
